01

FROKY VAPE

LIVE · 운영 중

베이프 제품 메뉴 사이트 + 관리자 백엔드. GitHub 토큰을 브라우저에 노출하지 않도록 Cloudflare Worker 프록시로 격리하고, Worker가 GitHub Contents API로 products.json을 직접 커밋(SHA 충돌 방지)하는 보안 아키텍처를 직접 설계. GSC 색인 분석과 sitemap·robots SEO까지 갖춰 실제 매장에서 운영 중.

ROLE기획 · 디자인 · 풀스택 · 백엔드 보안 설계 단독 구현
STACK
HTML/CSS/JSCloudflare WorkersGitHub Contents APIOAuth 2.0GitHub Pages
RESULT→ 브라우저에 토큰을 두지 않는 안전한 헤드리스 CMS 구조를 직접 구현 · 실서비스 운영 중
PROBLEM

제품 메뉴를 웹으로 운영하려면 상품 데이터 수정에 GitHub 토큰이 필요한데, 이를 브라우저에 두면 그대로 노출되는 보안 문제가 있었습니다.

APPROACH

토큰을 Cloudflare Worker 뒤에 숨기고 관리자는 X-Admin-Key 헤더로만 인증하게 설계했습니다. Worker가 GitHub Contents API로 products.json을 직접 커밋해 별도 DB·서버 없이 헤드리스 CMS를 구현하고, sitemap·robots·GSC로 검색 노출까지 챙겼습니다.

LEARNED

“프론트에 비밀을 두지 않는다”는 보안 원칙을 실제 운영 서비스에서 직접 구현하며, 인증·토큰 격리·SEO가 서비스 완성도를 좌우함을 배웠습니다.
